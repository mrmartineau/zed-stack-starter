# AGENTS.md

Guide for AI coding agents working in this repo. Humans should read [`README.md`](./README.md) first — this file assumes that context and focuses on the conventions an agent needs to make correct edits without breaking things.

## Stack at a glance

- **Runtime**: Cloudflare Workers (edge). `nodejs_compat` flag is on.
- **Frontend**: React 19 + [TanStack Router](https://tanstack.com/router) (file-based) + [TanStack Query](https://tanstack.com/query).
- **API**: [Hono](https://hono.dev/) mounted at `/api`, defined in `src/worker/hono.ts`.
- **DB**: Postgres via [Drizzle ORM](https://orm.drizzle.team/) + `@neondatabase/serverless` HTTP driver. Schema in `db/schema.ts`.
- **Auth**: [better-auth](https://www.better-auth.com/) (email/password). Server config in `src/lib/auth/server.ts`, React client in `src/lib/auth/client.ts`.
- **UI**: [ZUI](https://github.com/mrmartineau/zui) — class-based, no JS wrappers.
- **Build/lint/test**: Vite, Biome, Vitest.

## Where things live

```
db/
├── schema.ts            Single source of truth for tables. Edit here, then `bun run db:generate`.
└── client.ts            createDb(env) — neon-http drizzle client.
drizzle/                 Generated migration SQL. Commit these.
src/
├── components/          Reusable React components. ZUI classes, no styled-components.
├── constants.ts         Route paths (ROUTE_*), API paths (API_*), copy (CONTENT). Add new routes here too.
├── lib/
│   ├── auth/            better-auth server + client.
│   └── fetching/        Server-side helpers (getSession, profile queries).
├── routes/              File-based router. Folder names map to URL segments; `_prefix` = pathless layout.
│   ├── __root.tsx       App shell. Lives at every URL.
│   ├── _public/         Unauthenticated pages.
│   ├── _authed/         Pages behind auth. `route.tsx` redirects to /login if no session.
│   └── index.tsx        /
├── worker/
│   ├── index.ts         Worker entry.
│   ├── hono.ts          API routes. Add handlers here.
│   ├── env.ts           Typed bindings (WorkerEnv). Update when adding env vars.
│   ├── context.ts       requireRequestContext / createRequestContext.
│   └── profile.ts       Example endpoint shape — copy this for new resource files.
└── routeTree.gen.ts     **Generated.** Do not hand-edit.
```

## Hard rules

- **Never edit `src/routeTree.gen.ts`** — it's regenerated by the TanStack router Vite plugin.
- **Never edit files in `drizzle/` by hand** — regenerate with `bun run db:generate`.
- **Never edit `worker-configuration.d.ts`** — regenerate with `bun run cf-typegen`.
- **Don't hard-code route paths in JSX.** Use the `ROUTE_*` constants in `src/constants.ts`. When you add a top-level route, add the constant.
- **Don't introduce a new HTTP client or fetch wrapper.** Use plain `fetch` from the browser, or Drizzle directly on the server.
- **Don't add CSS-in-JS or new styling libraries.** ZUI classes + tokens only. If you need a new utility, add it to `src/styles.css`.
- **Don't bypass `requireRequestContext`** in authed Hono handlers. Returning a `Response` short-circuits — check `instanceof Response` and return it.
- **Imports** use the `@/` alias for `src/` (e.g. `import { ... } from "@/lib/..."`). Use it consistently.

## Recipes

### Add a route

1. Create `src/routes/<path>.tsx` (or `.../<segment>/<path>.tsx` for nested). Use `_authed/` for protected, `_public/` for explicitly public.
2. Export a `Route` via `createFileRoute("/exact-path")({...})`.
3. If it's a top-level destination linked from elsewhere, add a `ROUTE_*` constant in `src/constants.ts` and link via `<Link to={ROUTE_FOO}>`.
4. The route tree regenerates while `bun run dev` runs. Don't commit edits to `routeTree.gen.ts` separately from the route file.

Minimal route:

```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/widgets")({
  component: Widgets,
});

function Widgets() {
  return <div>Widgets</div>;
}
```

Authed route — drop into `src/routes/_authed/app/widgets.tsx`. `_authed/route.tsx` already handles the redirect-if-no-session in `beforeLoad`; don't duplicate it.

### Add an API endpoint

1. If the handler is small, add it inline in `src/worker/hono.ts`.
2. If it's a feature with multiple handlers, create `src/worker/<feature>.ts` exporting individual functions (mirror `profile.ts`), then wire them up in `hono.ts`.
3. Always read auth via `requireRequestContext(c)` (returns 401 `Response` on failure) or `createRequestContext(c)` (returns `null`-ish ctx for optional auth). Check `instanceof Response`.

```ts
// src/worker/widgets.ts
import type { Context } from "hono";
import { eq } from "drizzle-orm";
import { widgets } from "../../db/schema";
import { requireRequestContext } from "./context";
import type { WorkerEnv } from "./env";

export async function listWidgets(c: Context<{ Bindings: WorkerEnv }>) {
  const ctx = await requireRequestContext(c);
  if (ctx instanceof Response) return ctx;
  const rows = await ctx.db.select().from(widgets).where(eq(widgets.ownerId, ctx.user.id));
  return c.json(rows);
}
```

```ts
// src/worker/hono.ts
import { listWidgets } from "./widgets";
app.get("/widgets", listWidgets);
```

### Add or modify a database table

Full guide: [`docs/DATABASE.md`](./docs/DATABASE.md). Cliff notes:

1. Edit `db/schema.ts`. Match existing patterns — alphabetical fields, snake_case column names, explicit `references(..., { onDelete: ... })` on FKs, indexes/checks declared in the third arg of `pgTable`.
2. Run `bun run db:generate`. Inspect the new SQL in `drizzle/` — if you see a `DROP`, `ALTER ... DROP COLUMN`, or rename you didn't intend, **stop and ask the user**. drizzle-kit treats column renames as drop+create unless you tell it otherwise.
3. Hand-edit the SQL when needed: backfill before `SET NOT NULL`, add `USING` to type changes, split destructive changes into safe steps. After editing, do not re-run `db:generate` before migrating.
4. Commit `db/schema.ts` + new SQL file + updated `drizzle/meta/` together.
5. `bun run db:migrate` to apply locally. Don't run against staging/prod without explicit user request.

### Add a query

```ts
import { eq } from "drizzle-orm";
import { createDb } from "../../db/client";
import { widgets } from "../../db/schema";

const db = createDb(env);
const [widget] = await db.select().from(widgets).where(eq(widgets.id, id));
```

Inside Hono, prefer the `db` already on the request context — don't create a second client.

### Add a client-side data fetch

`QueryClient` is provided in `src/main.tsx`. Use `useQuery` directly:

```tsx
import { useQuery } from "@tanstack/react-query";

const { data, isLoading } = useQuery({
  queryKey: ["widgets"],
  queryFn: () => fetch("/api/widgets").then((r) => r.json()),
});
```

For session/profile data, prefer the helpers in `src/lib/fetching/user.ts` — don't duplicate the session fetch.

### Add a component

- Drop it in `src/components/`.
- Plain HTML + ZUI classes (`zui-button`, `zui-card`, `zui-input`, …).
- Tokens are CSS custom properties (`var(--space-md)`, `var(--color-...)`, `var(--step-2)`).
- Phosphor icons via `@phosphor-icons/react`.

### Add an environment variable

1. `.env` for local dev.
2. `wrangler.jsonc` `vars` (non-secret) or `bunx wrangler secret put NAME` (secret).
3. Type it on `WorkerEnv` in `src/worker/env.ts`.
4. `bun run cf-typegen` if it's a new binding type.

## Commands

| Command | When to run |
| --- | --- |
| `bun install` | After dependency change. |
| `bun run dev` | Local dev server. Keeps router tree + Vite HMR alive. |
| `bun run build` | Production build (Vite + tsc). |
| `bun run type-check` | Fast TS-only check. Run before declaring a task done. |
| `bun run check` | Biome lint + format with autofix. Run before committing. |
| `bun run test` | Vitest. |
| `bun run db:generate` | After editing `db/schema.ts`. |
| `bun run db:migrate` | Apply pending SQL to `DATABASE_URL`. |
| `bun run db:studio` | Inspect DB visually. |
| `bun run cf-typegen` | After changing `wrangler.jsonc` bindings. |
| `bun run deploy` | Build + `wrangler deploy`. Don't run without explicit user request. |

## Pre-handoff checklist

Before declaring a task done:

1. `bun run check` — Biome happy.
2. `bun run type-check` — TS happy.
3. `bun run test` — if you touched anything covered by a test or wrote one.
4. If you edited `db/schema.ts`: matching SQL file committed in `drizzle/`.
5. If you added a route: corresponding `ROUTE_*` constant where it's linked from JSX.
6. No edits to generated files (`routeTree.gen.ts`, `drizzle/*.sql`, `worker-configuration.d.ts`) made by hand.

## Things to ask before doing

- Destructive migrations (drop column, drop table, rename without `using`).
- Running `bun run deploy`.
- Running `bunx wrangler secret put` against a non-local environment.
- Adding a new top-level dependency.
- Switching the DB driver away from `@neondatabase/serverless`.
